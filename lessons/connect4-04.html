コネクトフォー LESSON 04 / 06

ゲームを完成させる

部品が出そろいました。列を選んで落とす、判定する、交代する——このくり返しでゲームループを組み、1局を最後まで走らせます。

🎯 このレッスンのゴール

  • ゲーム全体の流れ(ゲームループ)を設計できる
  • 盤面が満杯になったときの引き分け処理を実装する
  • 1局を自動対局させて、勝敗を発表する

STEP 1ゲームの流れを言葉で書く

1. 盤面を初期化。手番は黒から
2. くり返し:
     a. 落とせる列の一覧を作る
     b. 一覧が空なら → 引き分けで終了(満杯)
     c. どこかの列に石を落とす
     d. 落とした場所を起点に4つ並びをチェック → 並んでいれば勝者決定で終了
     e. 手番を交代

五目並べとほぼ同じ形ですが、「空きマスの一覧」の代わりに「落とせる列の一覧」を使う点が違いです。

STEP 2ゲームループを組む

今回も「落とせる列からランダムに選ぶ」対局にします。dropが返す(x, y)をそのままcheck_winに渡せるのが、これまでの設計の効いてくるところです。

コネクトフォーは全マスが埋まる(42マス)までに決着がつかないと引き分けです。4つ並べるには最低7手かかるので、ランダム同士だと、だいたい15〜30手くらいで決着することが多いはずです。

dropが (x, y) のタプルを返す設計にしておいたおかげで、check_win(board, x, y, turn) にそのまま渡せています。関数の戻り値を次の関数にそのまま渡せるように設計しておくと、ゲームループがすっきり書けます。

チャレンジ:100局トーナメント

ランダム同士で黒・白・引き分けの割合を調べてみましょう。

解答例を見る 
black_wins = white_wins = draws = 0

for i in range(100):
    board, winner = play_game()
    if winner == BLACK:
        black_wins += 1
    elif winner == WHITE:
        white_wins += 1
    else:
        draws += 1

print(f"黒 {black_wins} 勝 / 白 {white_wins} 勝 / 引き分け {draws}")

五目並べと同じく、先に打てる黒がわずかに有利な傾向が出やすいはずです。

STEP 3まとめ

  • コネクトフォーのゲームループは「落とせる列を選ぶ→落とす→判定→交代」のくり返し
  • 落とせる列がなくなったら引き分け
  • dropの戻り値をそのままcheck_winに渡せる設計が効いてくる

ゲームは完成。最終レッスンでは、ランダムより賢く打つAIを作ります。
