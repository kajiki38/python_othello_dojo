ゲームを完成させる

部品が出そろいました。列を選んで落とす、判定する、交代する——このくり返しでゲームループを組み、1局を最後まで走らせます。

🎯 このレッスンのゴール

  • ゲーム全体の流れ(ゲームループ)を設計できる
  • 盤面が満杯になったときの引き分け処理を実装する
  • 1局を自動対局させて、勝敗を発表する

STEP 1ゲームの流れを言葉で書く

1. 盤面を初期化。手番は黒から
2. くり返し:
     a. 落とせる列の一覧を作る
     b. 一覧が空なら → 引き分けで終了(満杯)
     c. どこかの列に石を落とす
     d. 落とした場所を起点に4つ並びをチェック → 並んでいれば勝者決定で終了
     e. 手番を交代

五目並べとほぼ同じ形ですが、「空きマスの一覧」の代わりに「落とせる列の一覧」を使う点が違いです。

STEP 2ゲームループを組む

今回も「落とせる列からランダムに選ぶ」対局にします。dropが返す(x, y)をそのままcheck_winに渡せるのが、これまでの設計の効いてくるところです。

Pythonには最初から使える道具箱(モジュール)がたくさん付属しています。import random で「乱数の道具箱」を読み込むと、random.choice(リスト)(=リストから1つ無作為に選ぶ)のように道具箱名.道具名の形で使えます。import はファイルの先頭に書くのが習慣です。

show_moves=False は「省略したときの値(デフォルト値)つき引数」。呼ぶときは play_game(show_moves=True) のように名前で指定できます。

True は常に成り立つ条件なので、while True: はわざと終わらないループです。きほんコースで注意した無限ループを、「決着したら return で抜ける」前提で制御して使う定番の型——ゲームのように終了条件が複数あってループの途中で決まる場面で活躍します。

not は True/False をひっくり返す演算子です。Pythonでは空のリストは False 扱いになるため、not cols は「落とせる列が1つもなければ」と読めます(同じ理屈で None も False 扱いです)。

コネクトフォーは全マスが埋まる(42マス)までに決着がつかないと引き分けです。4つ並べるには最低7手かかるので、ランダム同士だと、だいたい15〜30手くらいで決着することが多いはずです。

dropが (x, y) のタプルを返す設計にしておいたおかげで、check_win(board, x, y, turn) にそのまま渡せています。関数の戻り値を次の関数にそのまま渡せるように設計しておくと、ゲームループがすっきり書けます。

チャレンジ:100局トーナメント

ランダム同士で100局対戦させて、黒・白・引き分けの割合を調べましょう。集計係の関数 tally(winners) — 勝者のリスト(BLACK・WHITE・引き分けはNone)を受け取り、(黒の勝ち数, 白の勝ち数, 引き分け数)のタプルを返す — を下の専用エディタで完成させれば、トーナメントの集計が動き出します。スターターの black_wins = white_wins = draws = 0 は3つの変数をまとめて 0 にする書き方です。実行すると複数のテストデータで自動チェックされます。

解答例を見る 
import random

def play_game():
    board = make_board()
    turn = BLACK
    while True:
        cols = valid_columns(board)
        if not cols:
            return None
        col = random.choice(cols)
        x, y = drop(board, col, turn)
        if check_win(board, x, y, turn):
            return turn
        turn = opponent(turn)

def tally(winners):
    black_wins = white_wins = draws = 0
    for winner in winners:
        if winner == BLACK:
            black_wins += 1
        elif winner == WHITE:
            white_wins += 1
        else:
            draws += 1
    return black_wins, white_wins, draws

winners = [play_game() for _ in range(100)]
black, white, draws = tally(winners)
print(f"黒 {black} 勝 / 白 {white} 勝 / 引き分け {draws}")

五目並べと同じく、先に打てる黒がわずかに有利な傾向が出やすいはずです。「対局する関数」と「集計する関数」を分けておくと、次のレッスンでAI同士を戦わせるときも同じtallyがそのまま使い回せます。
