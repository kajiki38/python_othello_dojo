五目並べ LESSON 03 / 06

5つ並びの判定

五目並べの心臓部です。石を置いた場所から縦・横・斜めをたどって、同じ色が5つ以上連続しているかを調べます。「方向をたどって数える」——ゲームの盤面処理の定番ワザを学びます。

🎯 このレッスンのゴール

  • 「4つの軸(縦・横・斜め2方向)」という考え方を理解する
  • ある方向に何個連続しているかを数える count_line を作る
  • 勝敗判定 check_win を完成させる

STEP 18方向ではなく「4つの軸」でよい理由

盤面のまわりは全部で8方向ありますが(オセロの挟み判定では8方向すべてを使います)、五目並べは並びを数えるだけなので、反対向きの2方向はセットで1つの軸として扱えます。

横方向    : (1, 0)  ← → 反対は (-1, 0)
縦方向    : (0, 1)  ← → 反対は (0, -1)
斜め(右下): (1, 1)  ← → 反対は (-1, -1)
斜め(右上): (1, -1) ← → 反対は (-1, 1)

つまり4つの方向それぞれについて「置いた場所から前方向に数えた数」+「後ろ方向に数えた数」+「自分自身の1個」を合計すれば、その軸の連続数が分かります。

STEP 2一方向に数える count_line

まずは「片方向にだけ」同じ色が何個連続しているかを数える部品を作ります。盤外に出るか、違う色・空きマスに当たったら数えるのをやめます。

count_line がやっているのは、1マスずつ「歩く」だけです。nx, ny = x + dx, y + dy でまず隣の1マスへ移り、while で「盤の中」かつ「同じ色」が続く間だけ count を増やしながら nx += dx; ny += dy でさらに先へ進みます。違う色・空きマス・盤の外にぶつかった瞬間に止まります。(dx, dy) は進む向き——(1, 0) なら右、(-1, 0) なら左です。最初の1歩をループの前で踏んでいるのは、while が条件チェックから始まるループだから。ループに入る時点で nx, ny が「最初に調べたい隣のマス」を指している必要があり、スタート地点の自分の石まで数えてしまわないための工夫でもあります。

上のコードでは横一列に3つ並べたので、右方向は0個(その先に石がない)、左方向は2個(黒が2つ連続)という結果になったはずです。この2つを足して自分の1個を足すと「3」——実際に並んでいる数と一致します。

STEP 34つの軸すべてをチェック — check_win

count_line を前後2方向ぶん呼び出して、4つの軸すべてで判定すれば完成です。

check_win は「直前に置いた石」を起点に調べる設計です。盤面全体を毎回スキャンするより圧倒的に速く、64マス・81マスの盤面でも一瞬で判定できます。

チャレンジ:check_win を自分の手で完成させよう

心臓部の check_win を、下の専用エディタで自分の手で完成させましょう。count_line は用意してあるので、「4つの軸それぞれで 1 + 前方向 + 後ろ方向 を数えて、5以上なら True」を書くだけです。実行すると、縦・横・斜め2方向、そして「4つではまだ勝ちでない」ケースまで自動でチェックされます。

解答例を見る 
DIRECTIONS4 = [(1, 0), (0, 1), (1, 1), (1, -1)]

def count_line(board, x, y, dx, dy, color):
    count = 0
    nx, ny = x + dx, y + dy
    while 0 <= nx < SIZE and 0 <= ny < SIZE and board[ny][nx] == color:
        count += 1
        nx += dx
        ny += dy
    return count

def check_win(board, x, y, color):
    for dx, dy in DIRECTIONS4:
        total = 1 + count_line(board, x, y, dx, dy, color) + count_line(board, x, y, -dx, -dy, color)
        if total >= 5:
            return True
    return False

board = make_board()
for i in range(5):
    place(board, i + 2, i + 2, BLACK)
print_board(board)
print("斜めの5つ並び:", check_win(board, 4, 4, BLACK))

DIRECTIONS4 に (1, 1) と (1, -1) が含まれているおかげで、斜めもまったく同じコードでチェックできます。方向を「データ」として扱う設計の強みです。

STEP 4まとめ

  • 反対向きの2方向をペアにすれば、4つの軸だけで全方向をカバーできる
  • count_line = 一方向に連続する同じ色の数を数える部品
  • check_win = 4軸それぞれで「前方向+後ろ方向+自分」が5以上かを調べる
