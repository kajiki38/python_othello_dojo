LESSON 05 / 08

打てる場所を探す

「どこに置ける?」をコンピュータに全部探させます。合法手の一覧・石数のカウント・パスの判定——ゲーム完成に必要な残りの部品を一気に揃えましょう。

🎯 このレッスンのゴール

  • 打てる場所を全部リストアップする valid_moves を作る
  • 石の数を数える count_stones を作る
  • 「パス」をどう判定するかを理解する

STEP 1valid_moves — 全マスをチェック

今日作るのは、この「打てる場所探し」です。初期盤面なら黒が打てるのは4か所——これをコンピュータに探させます。

作戦は単純明快。64マス全部について「そこに置いたら1個でも裏返せる?」を聞いてまわるだけです。前回作った stones_to_flip がそのまま使えます。

初期盤面では黒も白も4か所ずつ。オセロを知っている人なら「たしかに!」という結果のはずです。if stones_to_flip(...) は「返ってきたリストが空でなければ」という意味でしたね。

STEP 2打てる場所を盤面に表示する

座標の数字だけでは分かりにくいので、打てる場所に * 印をつけて表示してみましょう。デバッグ(動作確認)にとても便利なテクニックです。

(x, y) in moves — 「このペアはリストに入ってる?」を1発で聞ける、Pythonの気持ちいい構文です。

STEP 3石を数える — count_stones

勝敗判定に必要な「石の数」。リストの count メソッドを使えばあっという間です。

+= は「足して代入」の省略形。total = total + ... と同じ意味です。空きマスも同じ関数で数えられるのがちょっと得した気分ですね。

STEP 4パスの判定

オセロでは、打てる場所が1つもないときパスになります。そして両者連続でパスならゲーム終了。判定は valid_moves の結果が空かどうかを見るだけです。

「白石が1つもなければ黒は何も挟めない」——自分の頭でルールを追いかけて、プログラムの出力と一致するか確かめるクセをつけると、デバッグ力がぐんぐん伸びます。

チャレンジ:どちらが「自由」か?

「打てる場所の数」は強さに直結する重要な指標です(着手可能数と呼びます)。初期盤面から黒が(2,3)、白が(2,2)、黒が(3,2)と進めた局面で、黒と白の打てる場所の数を比べて、多い方を「◯◯のほうが自由!」と表示するプログラムを書いてみましょう。

解答例を見る 
board = make_board()
place(board, 2, 3, BLACK)
place(board, 2, 2, WHITE)
place(board, 3, 2, BLACK)
print_board(board)

b = len(valid_moves(board, BLACK))
w = len(valid_moves(board, WHITE))
print(f"黒: {b} か所 / 白: {w} か所")
if b > w:
    print("黒のほうが自由!")
elif w > b:
    print("白のほうが自由!")
else:
    print("互角!")

len(リスト) でリストの長さが取れます。この「着手可能数」、最終レッスンのAI強化で再登場します。

STEP 5まとめ

  • valid_moves = 64マス総当たりで「置ける?」を確認するだけ
  • count_stones で石数、len(valid_moves(...)) で打てる場所の数
  • 打てる場所ゼロ → パス。両者連続パス → ゲーム終了
