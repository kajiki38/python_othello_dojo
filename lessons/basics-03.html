きほん LESSON 03 / 04

条件分岐

「黒のほうが多ければ黒の勝ち」「そのマスが空いていれば置ける」「盤の外なら調べない」——ゲームの判断は、すべてif(条件分岐)でできています。

🎯 このレッスンのゴール

  • if / elif / else で処理を分けられる
  • 比較演算子(== != > <= など)と and / or が使える
  • for と if を組み合わせて「条件に合うものを数える」が書ける

STEP 1if と else — もし〜なら

まずは勝敗判定から。石の数を比べて、メッセージを出し分けます。

if 条件: の条件が成り立てばインデントの中が実行され、成り立たなければ else: の中が実行されます。for と同じく、コロンとインデントがセットです。

STEP 2elif — 3つ以上に分ける

ところで、引き分けを忘れていませんか? 「勝ち・負け・引き分け」のように3つ以上に分けるときは elif(else if の略)を挟みます。

比較演算子まとめ: == 等しい / != 等しくない / > < より大きい・小さい / >= <= 以上・以下。「等しい」は = ではなく ==(= は変数への代入)——ここは間違えやすいので要注意!

STEP 3and / or — 条件を組み合わせる

ゲーム作りで何百回も書くことになる、超重要な判定があります。「その座標は盤の中?」です。x も y も範囲内、という2つの条件を and でつなぎます。

0 <= x < 8 は「0以上かつ8未満」を1つの式で書けるPythonの便利な記法です。盤の外を調べようとするとプログラムが壊れるので、この「盤内チェック」はオセロでも五目並べでもコネクトフォーでも必ず登場します。

or — どちらか一方でも成り立てば

and が「両方とも成り立てばTrue」なのに対して、or は「どちらか一方でも成り立てばTrue」です。たとえば「そのマスに石があるか?」は、黒(●)か白(○)のどちらかならYES、という判定になります。

and は日本語の「かつ」、or は「または」とそのまま対応します。複雑な条件は ( ) でグループにまとめると、意図が読みやすくなります(上の例の「x が端かつy が端 = 角」のように)。オセロで「角は特別扱い」といった判定に、この形がそのまま出てきます。

STEP 4for + if — 条件に合うものを数える

リストとループとifを組み合わせると、「盤面から黒石を数える」というゲームの実処理が書けるようになります。

チャレンジ:白石と空きマスも数えよう

STEP 4と同じ行データから、今度は白石("○")の数を変数 white_count に、空きマス(".")の数を変数 empty_count に数えて表示するプログラムを、下の専用エディタに書いてみましょう。for + if の練習です。実行すると自動でチェックされます。

解答例を見る 
row = ["●", "●", "○", ".", "○", "○", "○", "."]

white_count = 0
empty_count = 0
for i in range(8):
    if row[i] == "○":
        white_count = white_count + 1
    if row[i] == ".":
        empty_count = empty_count + 1

print(f"白石は {white_count} 個、空きマスは {empty_count} 個")

前のレッスンで出てきた row.count("○") を使えば1行で書けますが、「forとifで自力で数える」型はこの先ずっと使うので、両方できるようにしておきましょう。

STEP 5まとめ

  • if / elif / else で処理を分ける。条件のあとはコロンとインデント
  • 「等しい」は ==。=(代入)と間違えない
  • 0 <= x < 8 and 0 <= y < 8 — 盤内チェックは全種目共通の必修ワザ
  • for + if = 「条件に合うものを探す・数える」の基本形

最終レッスンは関数。ここまでの道具をまとめて「ゲームの部品」を作れるようになれば、きほんコースは卒業です。
