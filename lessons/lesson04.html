石をひっくり返す

オセロの心臓部、「挟んだ石を裏返す」ロジックを完成させます。このレッスンが本教材の山場。越えれば景色が変わります。

🎯 このレッスンのゴール

  • 「挟める石のリスト」を返す stones_to_flip を完成させる
  • ルールに従って石を置く place を完成させる
  • 実際に石がパタパタ裏返るのを自分のコードで確認する

STEP 1ルールをプログラムの言葉に翻訳する

オセロのルール「相手の石を挟んだら裏返せる」を、コンピュータが分かる手順に分解すると、こうなります。

  1. 置きたいマスから、8方向それぞれに1マスずつ進む
  2. 進んだ先が相手の石なら、リストに記録してさらに進む
  3. 相手の石の列の先に自分の石があれば、記録した石は全部「挟めた」
  4. 先が空きマスや盤の端なら、その方向は挟めない(記録は捨てる)

STEP 2まず1方向だけで考える

いきなり8方向は大変なので、右方向 (dx, dy) = (1, 0) だけで「挟める石集め」を書いてみます。前回の「直進walk」に、記録用リストを足しただけです。

whileの条件が3つ重なっているのがミソです。「盤の中」かつ「相手の石」の間だけ進む。抜けたあと、line に石が入っていて、かつ止まった場所が自分の石なら挟み成立です。

if line and ... の line は「リストが空っぽでない」という意味になります。空のリストはPythonでは「偽(False)」として扱われる——よく使うイディオムです。もうひとつ大事なのが条件の順番。and は左から調べ、左が偽なら右は見ません。盤内チェックを先に書いてあるおかげで、盤の外のマス(board[ny][nx])をうっかり読んで壊れることがないのです。

STEP 38方向まとめて — stones_to_flip 完成

あとはこれを for dx, dy in DIRECTIONS: で包んで、8方向分の結果を合体させるだけ。オセロエンジンで最重要の関数、完成させましょう。

flips += line は、これまで集めた flips の後ろに line の中身をまるごとつなげる書き方です(リスト版の足し算)。数の +=(count += 1 のような「足して代入」)と同じ感覚で、リストでは「連結して代入」になります。これで8方向ぶんの「挟めた石」が1つの flips にまとまります。

テスト結果を確認しましょう。初期盤面で黒が(2,3)に置くと(3,3)の白を挟める、(0,0)は何も挟めないので空リスト——期待通りに動いていますか?

STEP 4place — 置いて、裏返す

仕上げです。stones_to_flip の結果を使って、実際に盤面を書き換える place 関数を作ります。挟めない場所なら「置けないよ」と False を返して盤面を触りません。

not は True/False をひっくり返す演算子です。Pythonでは空のリストは False 扱いになるため、not flips は「1つも挟めない(flips が空)なら」と読めます。

白石が黒に変わり、置けない場所はちゃんと拒否される。おめでとうございます、オセロのルールがあなたのコードで動きました!

チャレンジ:連続対局

黒→白→黒…と交互に打ち進める「進行係」を関数にしましょう。play_moves(board, moves) は、座標リスト moves の手を黒から交互に置いていき、置けない手が出たらそこで中断して、置けた手数を返す関数です。break はループをその場で打ち切って外へ抜ける命令です(今回のように「失敗したら残りの手は打たない」に便利)。下の専用エディタに書いて実行すると、いろいろな手順のデータで自動チェックされます。

解答例を見る 
def stones_to_flip(board, x, y, color):
    if board[y][x] != EMPTY:
        return []
    flips = []
    for dx, dy in DIRECTIONS:
        line = []
        nx, ny = x + dx, y + dy
        while 0 <= nx < 8 and 0 <= ny < 8 and board[ny][nx] == opponent(color):
            line.append((nx, ny))
            nx, ny = nx + dx, ny + dy
        if line and 0 <= nx < 8 and 0 <= ny < 8 and board[ny][nx] == color:
            flips += line
    return flips

def place(board, x, y, color):
    flips = stones_to_flip(board, x, y, color)
    if not flips:
        return False
    board[y][x] = color
    for fx, fy in flips:
        board[fy][fx] = color
    return True

def play_moves(board, moves):
    """moves の座標を黒→白→黒→…と交互に置いていく。
    置けない手が出たらそこで中断し、置けた手数を返す"""
    turn = BLACK
    count = 0
    for x, y in moves:
        if not place(board, x, y, turn):
            break                     # 置けない手が出たら中断
        count += 1
        turn = opponent(turn)         # 置けたら手番交代
    return count

board = make_board()
n = play_moves(board, [(2, 3), (2, 2), (3, 2), (2, 4)])
print(f"{n} 手進みました")
print_board(board)

place の返り値 True/False をそのまま if の条件に使うのがポイント。moves の座標を差し替えて「どこなら置けるか」を頭で考えながら遊ぶのもおすすめです。次のレッスンで、それをコンピュータに探させる関数を作ります。

STEP 5まとめ

  • 挟み判定 = 「8方向ループ × 直進walk × 記録リスト」の組み合わせ
  • stones_to_flip は「裏返せる石のリスト」を返す。空リスト = 置けない
  • place は置く+裏返すを実行し、成功/失敗を True/False で返す
