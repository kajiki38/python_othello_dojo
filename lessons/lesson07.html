対戦AIを作る

「AI」と聞くと大げさですが、正体は手を選ぶ関数。ランダムAIから始めて「欲張りAI」を作り、AI同士を100局戦わせる実験まで一気にいきます。

🎯 このレッスンのゴール

  • AI = 「盤面を見て手を返す関数」という考え方を身につける
  • 関数を引数として渡せることを理解する(これが効く!)
  • ランダムAIと欲張りAIを作り、対戦実験で強さを比較する

STEP 1AIの正体は、ただの関数

オセロAIに必要な能力はたった1つ。「盤面と手番を渡されたら、打つ手を1つ返す」。つまりこういう形の関数です。

def 何とかAI(board, color, moves):
    # moves(打てる場所リスト)の中から1つ選んで…
    return (x, y)

一番シンプルなAI、ランダムAIはもう作れますね。

STEP 2AI同士を戦わせる装置を作る

ここでPythonの強力な機能を使います。関数は変数に入れたり、引数として渡したりできるのです。これを使えば「黒のAIと白のAIを受け取って対戦させる」装置が作れます。

play_match(ai_random, ai_random) — カッコをつけずに関数名だけを渡しているのに注目。ai_random() だと「実行した結果」が渡ってしまいます。関数名だけ = 関数そのものを渡す、です。

STEP 3欲張りAI — 一番たくさん取れる手を選ぶ

初めての「考えるAI」を作ります。作戦は人間の初心者と同じ、「いま一番たくさん裏返せる手を選ぶ」。stones_to_flip の長さを全部の手について調べて、最大のものを選びます。

Python上級者はこう書きます: return max(moves, key=lambda m: len(stones_to_flip(board, m[0], m[1], color)))。1行で同じ意味! 今は読めなくてOKですが、「maxは選び方(key)を指定できる」と覚えておくと格好いいコードが書けます。

STEP 4実験:欲張り vs ランダム、どっちが強い?

いよいよ科学の時間です。1局だけでは運もあるので、50局戦わせて勝率で判断します。実行前に予想してみてください——欲張りAIの勝率は何%くらいだと思いますか?

{wins / 50 * 100:.0f} の :.0f は書式指定で「小数点以下は表示しない(四捨五入)」という意味。66.0% ではなく 66% とすっきり表示できます。

おそらく勝率60〜75%前後になったはず。「たくさん取る」は思ったより圧勝ではないのがオセロの面白いところ。序盤に取りすぎると、かえって相手に打てる場所を与えてしまうのです。この弱点を突く方法は、最終レッスンで。

チャレンジ:角が大好きなAI

オセロの格言「角を取れ」。角(0,0)(7,0)(0,7)(7,7)は絶対に裏返されない最強のマスです。「打てる場所に角があれば必ず角に打ち、なければ欲張りAIと同じ動きをする」ai_corner を、下の専用エディタで完成させましょう。実行すると「角と大量取りのどちらを選ぶ?」などいろいろな盤面で自動チェックされます。合格したら、STEP 4 のエディタで play_match(ai_corner, ai_random) を50局試して、欲張りAIより強いか実験してみてください。

解答例を見る 
CORNERS = [(0, 0), (7, 0), (0, 7), (7, 7)]

def ai_greedy(board, color, moves):
    best_move = moves[0]
    best_count = 0
    for x, y in moves:
        n = len(stones_to_flip(board, x, y, color))
        if n > best_count:
            best_count = n
            best_move = (x, y)
    return best_move

def ai_corner(board, color, moves):
    """角に打てるなら角、そうでなければ欲張りAIと同じ手を選ぶ"""
    for corner in CORNERS:
        if corner in moves:      # 角に打てるなら迷わず角!
            return corner
    return ai_greedy(board, color, moves)   # それ以外は欲張り

board = make_board()
moves = valid_moves(board, BLACK)
print("打てる場所:", moves)
print("ai_corner の選択:", ai_corner(board, BLACK, moves))

すでにあるAI(ai_greedy)を部品として呼び出しているのがポイント。AIの改造は「部品の組み合わせ」でどんどん進化させられます。50局対戦させると、ランダム相手の勝率は欲張りAIをさらに上回るはずです。

STEP 5まとめ

  • オセロAI = (board, color, moves) → 手 を返すただの関数
  • 関数は引数として渡せる。だから play_match(AI1, AI2) が書ける
  • 強さは大量対局の勝率で測る。1局の勝ち負けで判断しない

最終レッスンでは「マスの価値」という考え方でAIを本格的に強くします。
