五目並べ LESSON 04 / 06

ゲームを完成させる

部品はすべて揃いました。手番を交代しながら石を置き続け、5つ並んだら終了、誰も置けなくなったら引き分け——ゲームループを組んで1局を最後まで走らせます。

🎯 このレッスンのゴール

  • 空いているマスの一覧を作る empty_cells を作る
  • ゲーム全体の流れ(ゲームループ)を設計できる
  • 1局を自動対局させて、勝敗を発表する

STEP 1ゲームの流れを言葉で書く

オセロのときと同じく、まず日本語で整理します。

1. 盤面を初期化。手番は黒から
2. くり返し:
     a. 空いているマスの一覧を作る
     b. 空きマスがなければ → 引き分けで終了
     c. どこかに石を置く
     d. いま置いた場所を起点に5つ並びをチェック → 並んでいれば勝者決定で終了
     e. 手番を交代

オセロにあった「パス」がありません。五目並べは打てる場所がある限り必ず打てるので、その分ロジックがすっきりします。

STEP 2空いているマスを集める

「どこに置くか」を選ぶには、まず空きマスの一覧が必要です。盤面を二重ループでなめて集めましょう。

cells[:5] はスライスという書き方で、「リストの最初の5個だけ」を取り出します。空きマスは数十個あるので全部表示するとあふれてしまう——そんなときに先頭だけのぞき見できる便利な記法です。

STEP 3ゲームループを組む

今回は「打てる場所からランダムに選ぶ」対局にします。random.choice で空きマスを1つ選び、置いて、勝敗をチェックして、交代——これをくり返します。

実行するたびに違う棋譜が流れます。9×9盤(81マス)はランダム同士でもわりとすぐ5つ並びが生まれるので、数手〜数十手で決着することが多いはずです。

check_win を毎手番、置いた直後にだけ呼んでいる点がポイントです。「置く前に全部チェック」ではなく「置いた場所だけ調べれば十分」——これがcheck_winを軽量に保つ設計のコツでした(前回のレッスンの復習)。

チャレンジ:100局トーナメントの集計係

ランダム同士だと黒と白、引き分け、どのくらいの割合になるでしょうか? それを調べる集計係として、勝者を並べたリスト(例: [BLACK, WHITE, BLACK, None])を受け取り、(黒の勝ち数, 白の勝ち数, 引き分け数) を返す count_results 関数を下の専用エディタに書きましょう。実行すると複数のテストデータで自動チェックされます。完成したら、STEP 3 の play_game を100回呼んだ結果を渡せば、そのまま成績発表に使えます。

解答例を見る 
def count_results(winners):
    black_wins = white_wins = draws = 0
    for winner in winners:
        if winner == BLACK:
            black_wins += 1
        elif winner == WHITE:
            white_wins += 1
        else:
            draws += 1
    return black_wins, white_wins, draws

sample = [BLACK, WHITE, BLACK, None]
b, w, d = count_results(sample)
print(f"黒 {b} 勝 / 白 {w} 勝 / 引き分け {d}")

本番のトーナメントは、空のリストを用意して play_game() を100回くり返し、返ってきた勝者を append でためてから count_results に渡すだけ。やってみると、先に置く黒がわずかに有利な傾向が出やすいはずです。五目並べは実際に「先手必勝」が理論的に証明されているゲームとして知られています。

STEP 4まとめ

  • 五目並べのゲームループは「置く→判定→交代」のくり返し。パス処理が不要な分シンプル
  • 空きマスがなくなったら引き分け
  • 「置いた直後にだけ判定する」設計で、判定処理を軽く保てる

ゲームは完成。最終レッスンでは、ランダムより賢く打つAIを作ります。
