五目並べ LESSON 06 / 06

AIをもっと強く

前回作った ai_block には、実は人間が簡単に勝ててしまう「必勝パターン」があります。その正体ダブルリーチを突き止めて、1手先を読んで封じる強化AIを作りましょう。

🎯 このレッスンのゴール

  • 1手読みAIの弱点「ダブルリーチ」を理解する
  • リーチの数を数えて、ダブルリーチを「狙う」「先回りして防ぐ」AIを作る
  • 候補を絞って高速化する、というAI実装の定番テクニックを知る

STEP 1ai_block はなぜ負ける?

ai_block は「相手の即勝ち手(あと1手で5連になる場所)」しか防ぎません。では、こんな盤面ではどうでしょう。黒(相手)が両端の空いた3連を作っています。

結果を見てください。3連の段階では黒の即勝ち手は None — つまり ai_block の防御センサーには何も映っていません。だから白はランダムな場所に打ってしまい、次に黒が3連を4連に伸ばした瞬間、勝ち手が2つ(ダブルリーチ)になって手遅れです。何回か実行して、白が端を守らないことを確かめてみましょう。

1手読みの壁: 「今すぐ勝てる/負ける」しか見ないAIは、「次の手で必勝形を作られる」ことに気づけません。強くするには、1手先の形まで読む必要があります。

STEP 2リーチを「数える」

ダブルリーチを見抜く道具を作ります。find_winning_move は勝ち手を「1つ見つけたら終わり」でしたが、こんどはいくつあるかを数えます。おなじみ「試しに置く→判定→戻す」です。

3連は 0、両端の空いた4連は 2。リーチ数が2以上 = 相手は片方しか防げない = 事実上の勝ちです。この「2」を検出できれば、狙うことも防ぐこともできます。

STEP 3ダブルリーチを「狙う」「先回りして防ぐ」

部品はそろいました。ai_block に2つの読みを追加します。①自分がダブルリーチを作れる手があれば打つ(次のターンで勝ち確定)。②相手がダブルリーチを作れる場所があれば、先回りして自分がそこに置く。

今度は白が (1, 4) か (5, 4) — 3連の端 — を選んだはずです。「相手がここに置いたらリーチが2つになる」と1手先を読んで、必勝形ができる前に芽を摘みました。

②のループでは board[y][x] = opponent(color) と、相手の石を置いて試しています。「相手の立場になって盤面を読む」——人間の対局と同じ発想が、そのままコードになっています。

STEP 4高速化 — 候補を絞る

STEP 3 のAI、実は計算量に難ありです。「全空きマス × 全空きマス」で調べるので、序盤は 79 × 78 ≒ 6000 回も勝敗判定を回します。でも考えてみると、意味のある手はほぼ「すでにある石の近く」だけ。候補をそこに絞りましょう。

ここで初めて set(セット=集合)が登場しました。リストとよく似ていますが、同じ値を重複して持たないのが特徴です。set() で空の集合を作り、.add(値) で足していきます(すでにある値を足しても増えません)。石の周囲を調べると同じ空点を何度も拾ってしまうので、自動で重複を消してくれる set がぴったり。最後に list(cells) で、いつものリストに戻しています。

結果は79個 → 12個。調べる数が数分の一になれば、掛け算で効いて数十倍速くなります。「正しく動くものを作ってから、無駄を削って速くする」——これもプログラミングの王道の手順です。

STEP 5対戦実験:先読みAI vs 勝ち・防御AI

絞り込み版 ai_strong を完成させて、前回の ai_block と20局対戦させましょう(先手後手は1局ごとに交代)。なお、ランダム要素のないAIどうしだと毎回まったく同じ棋譜になってしまうため、play_match は各対局の最初の2手をランダムに打ってから実力勝負に入る作りにしてあります。※数秒かかることがあります。

圧勝のはずです。読みの深さがたった1手違うだけで、これだけの差がつきます。

チャレンジ:攻めをさらに鋭く

いまのAIは「ダブルリーチが作れないときは、ただ中央寄りに置く」だけ。ここを「せめてリーチ(勝ち手1つ)を作れる手を優先する」に改造した ai_strong2 を、下の専用エディタで完成させましょう。n >= 2 のループの後に、n >= 1 のループを足すだけです。実行すると、勝ち・防御・先回りといったこれまでの読みが壊れていないかまで、複数の盤面でまとめて自動チェックされます。

解答例を見る 
def near_cells(board):
    cells = set()
    for y in range(SIZE):
        for x in range(SIZE):
            if board[y][x] != EMPTY:
                for dy in (-1, 0, 1):
                    for dx in (-1, 0, 1):
                        nx, ny = x + dx, y + dy
                        if 0 <= nx < SIZE and 0 <= ny < SIZE and board[ny][nx] == EMPTY:
                            cells.add((nx, ny))
    return list(cells)

def count_reaches(board, color):
    count = 0
    for x, y in near_cells(board):
        board[y][x] = color
        if check_win(board, x, y, color):
            count += 1
        board[y][x] = EMPTY
    return count

def ai_strong2(board, color, moves):
    win_move = find_winning_move(board, color, moves)
    if win_move:
        return win_move
    block_move = find_winning_move(board, opponent(color), moves)
    if block_move:
        return block_move
    center = SIZE // 2
    cand = near_cells(board)
    if not cand:
        return (center, center)
    cand.sort(key=lambda m: (m[0] - center) ** 2 + (m[1] - center) ** 2)
    for need in (2, 1):                  # まずダブルリーチ、なければ単リーチ
        for x, y in cand:
            board[y][x] = color
            n = count_reaches(board, color)
            board[y][x] = EMPTY
            if n >= need:
                return (x, y)
        if need == 2:                    # ダブルリーチ狙いの直後に防御を挟む
            for x, y in cand:
                board[y][x] = opponent(color)
                n = count_reaches(board, opponent(color))
                board[y][x] = EMPTY
                if n >= 2:
                    return (x, y)
    return cand[0]

常にリーチを作り続けるAIは、相手に防御を強制し続けます。攻め続けている限り、相手は自分の形を作る暇がない——五目並べの「先手の理」をAIに教えたことになります。ai_strong2 と ai_strong を play_match で戦わせて、勝率がどう変わるか確かめてみましょう。
