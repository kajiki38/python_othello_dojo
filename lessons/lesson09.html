発展: 2手先を読むAI

Lesson 7 で作った ai_master は、実は「打った直後の evaluate」しか見ていません。相手がどう返してくるかまで読む——本格AIの中心アイデア「ミニマックス法」で、ai_master を超えるAIを作りましょう。

🎯 このレッスンのゴール

  • 「打った直後」しか見ない評価の限界を、実際の罠局面で体感する
  • 「自分の番は max、相手の番は min」——ミニマックスの考え方を理解する
  • 新しい武器再帰(自分を呼ぶ関数)を使って minimax_score を実装する
  • ai_minimax を完成させて、ai_master に勝ち越すことを確かめる

このページのウィジェットには、Lesson 7 までに作った全部品(place / valid_moves などのルール一式に加えて WEIGHTS / evaluate / copy_board / ai_master / play_match)が読み込み済みです(📦マーク)。

STEP 1「1手先」の限界 — ai_master が罠にはまる

Lesson 7 の最後に研究テーマとして挙げた「2手先読み」。なぜそれが必要なのか、まず ai_master が実際に罠にはまる局面を見てもらいます。序盤を10手すすめた局面で、黒番の ai_master に手を選ばせます。さらに各候補手について、「打った直後」の評価と、白にも ai_master として最善の返しを打たせた後の評価を並べてみましょう。

表を見てください。ai_master が選んだ (5, 0) は「直後」の評価 30 で堂々の1位。ところが白の最善の返しは角 (7, 0)——返された後の評価は -50 まで暴落します。いっぽう (2, 5) は直後 29 とたった1点差の2位なのに、返された後も 21 でトップ。1点の欲に目がくらんで角を献上してしまうわけです。

1手読みの壁: 「打った直後の点数」がどんなに良くても、相手の返しの一手で台無しになることがあります。手の本当の価値は「打った直後」ではなく「相手に返された後」で測らなければいけません。

STEP 2相手は一番イヤな返しをしてくる

そこで発想を変えます。「自分の手の価値 = 相手が最善(こちらに最悪)の返しをしてきた後の盤面の価値」。相手が甘い手を打ってくれることを祈るのではなく、一番イヤな返しをされる前提で手を選ぶのです。「一番イヤ = こちらの評価が最小になる返し」なので、Python では min() で書けます。

各手の価値が、STEP 1 の表の「返しの後」の列とぴったり同じ数字になりましたね。オセロは相手が得をした分だけ自分が損をするゲームなので、「白にとって最善の返し」と「黒にとって最悪の返し」は同じ手なのです。そして新しいものさしは、罠の (5, 0) ではなくちゃんと (2, 5) を選びました。

ここに出てきた2つの道具に注目。相手の番は min()(こちらに一番イヤな手を選んでくる)、自分の番は max(一番良い手を選ぶ)。この「max と min を交互に取りながら先を読む」方法をミニマックス法と呼びます。1950年代から現代のチェス・将棋AIまで、対戦ゲームAIの土台であり続けている考え方です。

STEP 3再帰 — 関数が自分を呼ぶ

STEP 2 は「2手先」まででしたが、3手先・4手先はどうしましょう? ループを二重・三重に増やす…のは書いていられません。ここで新しい武器、再帰の出番です。まずはミニマックスを離れて、いちばん小さな例から。関数が自分自身を呼ぶとどうなるか、実行して確かめてください。

関数が自分自身を呼ぶ書き方を再帰といいます。①必ず「止まる条件」を先に書く ②呼ぶたびに問題を小さくする(n - 1、depth - 1)——この2点さえ守れば怖くありません。ミニマックスは「depth 手先まで読む」を「1手進めて depth-1 手先まで読む」に置き換える、再帰の代表例です。

この型をミニマックスに当てはめます。「盤面を depth 手先まで読んで採点する関数」minimax_score は——止まる条件は「depth == 0(読み切った)か、打てる手がない」でそのときは evaluate で採点。それ以外は1手ずつ試し打ちして、残り depth-1 手の採点を自分自身に任せる。最後に自分の番なら max、相手の番なら min を取るだけです。

STEP 2 の手書き二重ループと同じ数字が出たはずです。でも決定的な違いがあります。score_min_reply は2手先専用でしたが、minimax_score は depth を増やすだけで何手先でも読めます。二重ループを三重・四重に書き直す必要はありません——深くもぐるのは再帰が勝手にやってくれます。

STEP 4ai_minimax 完成 — ai_master に挑む

仕上げです。「各候補手を試し打ちして、minimax_score が一番高い手を選ぶ」AIを作ります。骨組みは ai_master とまったく同じで、ものさしが evaluate から minimax_score に変わっただけです。そして本家 ai_master と20局対戦させます。対戦には Lesson 7 の STEP 4 で学んだ「序盤ランダム」方式(最初の4手をランダムに打って展開をバラけさせる)をここでも使います——決定的AI同士の対決だからこそ、でしたね。※2手読みは1手あたり約10ミリ秒。20局でも数秒で終わります。

どうでしたか? 運にも左右されますが、7〜8割は勝てるはずです(こちらの実験では50局で38勝11敗1分でした)。注目してほしいのは、ai_minimax が使っている評価関数は ai_master とまったく同じ evaluate だということ。「相手の返しまで読む」——それだけで、これだけの差がつくのです。

depth=2 を大きくすれば、さらに先まで読めます。ただし読む局面の数は掛け算で増えていくので、1手増やすごとに何倍も遅くなります。「どこまで深く読むか」は強さと速さのトレードオフ——本物のゲームAI開発でも永遠のテーマです。

チャレンジ:評価関数を強くする — 角ボーナス

ミニマックスの読みの底では、相変わらず evaluate が盤面を採点しています。つまり評価関数が賢くなれば、読み全体が賢くなるということ。オセロでは、角に置いた石は絶対にひっくり返されない「確定石」です。WEIGHTS は「角のマスは価値が高い」ことしか知らないので、確定石のうまみをボーナスとして上乗せしましょう。evaluate の点数に、4つの角を調べて自分の石なら +25、相手の石なら −25 する evaluate2 を作り、それで読む ai_minimax2 を完成させてください(minimax_score2 は完成品を用意したのでそのまま使えます)。実行するといろいろな盤面で自動チェックされます。

解答例を見る 
def evaluate2(board, color):
    score = evaluate(board, color)
    opp = opponent(color)
    for x, y in [(0, 0), (7, 0), (0, 7), (7, 7)]:
        if board[y][x] == color:
            score += 25
        elif board[y][x] == opp:
            score -= 25
    return score

def minimax_score2(board, color, turn, depth):
    moves = valid_moves(board, turn)
    if depth == 0 or not moves:
        return evaluate2(board, color)
    scores = []
    for x, y in moves:
        test = copy_board(board)
        place(test, x, y, turn)
        scores.append(minimax_score2(test, color, opponent(turn), depth - 1))
    return max(scores) if turn == color else min(scores)

def ai_minimax2(board, color, moves):
    best_move, best_score = None, None
    for x, y in moves:
        test = copy_board(board)
        place(test, x, y, color)
        s = minimax_score2(test, color, opponent(color), 1)
        if best_score is None or s > best_score:
            best_move, best_score = (x, y), s
    return best_move

# 動作チェック
board = make_board()
board[0][0] = BLACK
print("evaluate :", evaluate(board, BLACK))
print("evaluate2:", evaluate2(board, BLACK), "(evaluate より 25 大きくなれば成功)")
print("ai_minimax2 の選択:", ai_minimax2(make_board(), BLACK, valid_moves(make_board(), BLACK)))

合格したら STEP 4 のエディタに evaluate2 / minimax_score2 / ai_minimax2 を貼り込んで、play_match_varied(ai_minimax2, ai_minimax) の20局対戦へ。ボーナスの「25」を 10 や 50 に変えると勝率はどう動くでしょう? 評価関数のチューニングと読みの深さ——2つの改造軸を手に入れたあなたは、もう立派なゲームAI研究者です。

STEP 5まとめ — この型はどのゲームでも通用する

  • 1手先だけの評価は、相手の「返しの一手」で崩れる(角の献上)
  • 手の価値は「相手が一番イヤな返しをしてきた後」で測る——自分の番は max、相手の番は min
  • 再帰のルールは2つ: 止まる条件を先に書く/呼ぶたびに問題を小さくする
  • 同じ evaluate でも、読みを1手深くするだけで ai_master に勝ち越せる

今日作った minimax_score をもう一度見てください。オセロ固有の知識は evaluate と valid_moves の中に閉じ込められていて、ミニマックス本体には「max と min を交互に取る」ことしか書かれていません。つまり「評価関数+ミニマックス」の型は、五目並べでもコネクトフォーでも、どのゲームでもそのまま通用します。他の種目の発展レッスンで、ぜひ確かめてみてください。仕上げはAI工房へ——ai_minimax を持ち込んで、歴代の内蔵AIを蹴散らしましょう!
