4つ並びの判定

五目並べで作った「4つの軸を数える」考え方が、ここでもほぼそのまま使えます。違うのは並べる数(5→4)と、盤面が長方形なので範囲チェックをWIDTH・HEIGHTで分けて書く点だけです。

🎯 このレッスンのゴール

  • 長方形の盤面に合わせて範囲チェックを書き直せる
  • 4方向の連続数を数える count_line を作る
  • 勝敗判定 check_win を完成させる

STEP 1正方形と長方形、範囲チェックの違い

五目並べでは盤面が9×9の正方形だったので、xもyも同じSIZEで範囲チェックできました。コネクトフォーは横7×縦6なので、xは 0 <= x < WIDTH、yは 0 <= y < HEIGHT と別々にチェックする必要があります。この式がこのあとcount_lineの中にそのまま登場します。

STEP 2一方向に数える count_line

五目並べのcount_lineとほぼ同じですが、範囲チェックをWIDTH・HEIGHTそれぞれに対応させます。

count_line がやっているのは、1マスずつ「歩く」だけです。nx, ny = x + dx, y + dy でまず隣の1マスへ移り、while で「盤の中(0 <= nx < WIDTH かつ 0 <= ny < HEIGHT)」かつ「同じ色」が続く間だけ、count を増やしながら nx += dx; ny += dy でさらに先へ進み続けます。違う色・空きマス・盤の外にぶつかった瞬間に while が止まり、そこまでの連続数を返します。(dx, dy) は進む向き——(1, 0) なら右、(0, 1) なら下、(1, 1) なら右斜め下です。

STEP 34つの軸すべてをチェック — check_win

片方向を数えられたので、あとは反対向きと足し合わせるだけ。ある軸の連続数は「自分の石(1個)＋その向きに続く数＋逆向きに続く数」で求まります。だから 1 + count_line(…, dx, dy, …) + count_line(…, -dx, -dy, …)。(dx, dy) の逆向きは (-dx, -dy) です。これを4つの軸(縦・横・斜め2方向)で調べ、どれかが4以上なら勝ち——並べる数を「5」から「4」に変えるだけで、判定の組み立てはそのまま通用します。

DIRECTIONS4の中身は五目並べと完全に同じ4方向です。「縦・横・斜め2方向に並べたら勝ち」という判定ロジックそのものはゲームが変わっても再利用できる——これがこの2軸(4方向スキャン)方式の強みです。

チャレンジ:縦に4つ並べて確認しよう

同じ列に黒石を4回落として、縦の4つ並びでcheck_winがTrueになることを確認してみましょう。

解答例を見る 
board = make_board()
for i in range(3):
    x, y = drop(board, 3, BLACK)
print(check_win(board, x, y, BLACK))   # False (3個)

x, y = drop(board, 3, BLACK)
print(check_win(board, x, y, BLACK))   # True (4個)

DIRECTIONS4の(0, 1)(縦方向)が自動でこの並びを見つけてくれます。方向をリストで管理しているおかげで、追加のコードは一切不要でした。

STEP 4まとめ

  • 長方形の盤面では、xをWIDTHで、yをHEIGHTで別々に範囲チェックする
  • 判定ロジック(4軸スキャン)そのものはゲームが変わっても使い回せる
  • 並べる数だけをゲームに合わせて変える(オセロ:8方向裏返し / 五目並べ:5つ / コネクトフォー:4つ)
