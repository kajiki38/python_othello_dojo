対戦AIを作る

コネクトフォー道場、最終レッスンです。五目並べで学んだ「勝てる手を逃さず、負けそうな手を防ぐ」という考え方を、列を選ぶAIとして実装します。

🎯 このレッスンのゴール

  • 「試しに落としてみて判定し、元に戻す」テクニックを列バージョンで使う
  • 勝ち・防御を読む ai_block を作る
  • ランダムAIと対戦させて強さを実験する

STEP 1ランダムAI(基準点)

STEP 2「試しに落としてみる」— 列バージョン

五目並べでは座標に直接置いて調べましたが、コネクトフォーはdropを使って「試しに落として」から調べます。next_rowで落ちる場所(y)を先に確認しておくのがポイントです。

STEP 3勝ち + 防御 — ai_block

五目並べのai_blockと同じ組み立てです。まず自分の勝ち手、なければ相手の勝ち手(=防ぐべき手)、それもなければランダム。

win_col is not None の書き方に注目してください。列の番号は0もあり得るので、if win_col: と書くと0列目が「勝てる手なし」と誤判定されるバグになります。「値があるかどうか」を調べるときはis not Noneを使う——地味ですが重要な習慣です。

STEP 4実験:防御AI vs ランダムAI

関数を「渡す」: play_match(ai_block, ai_random) は、ai_block() と実行するのではなく、ai_block という関数そのものを引数として渡しています。play_match の中では受け取った関数を ai = ai_black if turn == BLACK else ai_white で手番に応じて選び、ai(board, turn, cols) の形で呼び出します。Pythonでは関数も値の一種なので、こうして「どのAIで戦わせるか」を差し替えられるのです。次のレッスンで作る強いAIも、同じ play_match にそのまま渡して試せます。

「勝ち」と「防御」の2つのルールだけで、ランダムAIに大きく差をつけられるはずです。

チャレンジ:中央列を優先するAI

コネクトフォーの定石は「中央の列(3列目)ほど多くの4つ並びに関われるので価値が高い」。ai_blockを改造して、勝ち・防御がなければ中央に近い列を優先するAIを作ってみましょう。

解答例を見る 
def ai_center(board, color, cols):
    win_col = find_winning_col(board, color, cols)
    if win_col is not None:
        return win_col
    block_col = find_winning_col(board, opponent(color), cols)
    if block_col is not None:
        return block_col
    center = WIDTH // 2
    return min(cols, key=lambda c: abs(c - center))

wins = losses = draws = 0
for i in range(50):
    winner = play_match(ai_center, ai_random)
    if winner == BLACK: wins += 1
    elif winner == WHITE: losses += 1
    else: draws += 1
print(f"中央志向AI: {wins}勝 {losses}敗 {draws}分")

key=lambda c: abs(c - center) は「各列 c を『中央からの距離』に変換してから比べる」指定です(abs は絶対値=符号を取った距離)。min がそのいちばん小さい=中央にいちばん近い列を選びます。lambda は「その場で作る名前のない小さな関数」で、今は『各要素を点数に変える式』と読めればOK。「まず勝ち手を探す→防御→評価値で選ぶ」というAI設計の型そのものは、ゲームが変わっても使い回せる——これがオセロ道場3種目を通じて一番伝えたかったことです。

STEP 5まとめ

  • 列を扱うAIでも「試しに落とす→判定→元に戻す」の考え方は同じ
  • is not Noneで「0」と「なし」を正しく区別する
  • 強さは大量対局の勝率で測る
