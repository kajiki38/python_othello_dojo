きほん LESSON 04 / 04

関数

きほんコース最終回。ゲームのプログラムは「盤面を作る」「石を置く」「勝ちを判定する」…という部品(関数)の組み合わせでできています。部品の作り方を覚えたら、いよいよ好きな種目へ!

🎯 このレッスンのゴール

  • def で関数を定義し、引数を渡して呼び出せる
  • return で結果を返し、受け取って使える
  • タプル (x, y) で複数の値をまとめて扱える

STEP 1def — 処理に名前をつける

関数とは「処理のかたまりに名前をつけたもの」。一度作れば、何度でも一言で呼び出せます。

def 関数名(引数): で定義して、インデントの中に処理を書きます。name のような引数は「呼び出すときに渡すデータの受け口」です。

STEP 2return — 結果を返す

表示するだけでなく、計算結果を呼び出し元に返すのが関数の本領です。前のレッスンで書いた勝敗判定を、部品化してみましょう。

同じ判定を3回書かずに済みました。「2回以上書きそうな処理は関数にする」——これがプログラムをきれいに保つ鉄則です。ちなみに """...""" は関数の説明メモ(docstring)で、書いておくと後で読み返すときに助かります。

STEP 3タプル — 複数の値をまとめて返す

ゲームでは「座標 (x, y)」のように2つセットの値を頻繁に扱います。カンマで並べた値の組をタプルと呼び、関数からまとめて返すこともできます。

2か所、初めて出た書き方があります。ひとつは //。ふつうのわり算 / は 9 / 2 = 4.5 と小数になりますが、マスの番号は整数でないと board[4.5] のように使えません。// は小数点以下を切り捨てて整数を返すわり算なので、9 // 2 = 4(中央のマス番号)がそのまま得られます。

もうひとつは for mx, my in moves:。moves の中身はタプル (2, 3) なので、取り出すたびに自動で mx = 2, my = 3 に分解(アンパック)されます。for m in moves: と書いて m[0] / m[1] で取り出すこともできますが、名前を付けて分解したほうがぐっと読みやすくなります。

この形、あとで何度も見ます: for mx, my in moves: のように「座標のリストを回す」書き方は、オセロの合法手探し・五目並べのAI・コネクトフォーの判定…あらゆる場面で登場します。ここで手に馴染ませておくと後がラクです。

STEP 4部品を組み合わせる

仕上げに、どの種目でも実際に使う関数 opponent(相手の色を返す)を作って、手番交代を再現してみましょう。関数 × ループの合わせ技です。

この opponent は、オセロ・五目並べ・コネクトフォーの3種目すべてにそのままの形で登場する本物の部品です。あなたはもう、ゲームエンジンの最初の1個を書き上げました。

1行で書く書き方(条件式): opponent の if/else は、次のように1行にまとめられます。

def opponent(color):
    return WHITE if color == BLACK else BLACK

Aの値 if 条件 else Bの値 は「条件が成り立てば A、そうでなければ B」を返す省略記法(条件式)です。各種目のエンジンではこの1行版がよく使われるので、見たら「あの if/else の短縮形だ」と思い出せるようにしておきましょう。

チャレンジ:盤内チェックを関数に

前のレッスンで書いた「盤内チェック」を、is_inside(x, y) という関数にして、下の専用エディタで完成させましょう。8×8の盤として、盤の中なら True、外なら False を返すようにします(比較式の結果はそのまま return できます)。実行すると、いろいろな座標で自動チェックされます。

解答例を見る 
def is_inside(x, y):
    return 0 <= x < 8 and 0 <= y < 8

print(is_inside(3, 5))    # True
print(is_inside(8, 0))    # False
print(is_inside(-1, 4))   # False

比較式の結果(True / False)をそのまま返せば、if を書かなくても1行で完成します。この関数も、各種目の「盤の外をたどらない」処理でそのまま活躍します。

STEP 5まとめ — きほんコース修了!

  • def 名前(引数): で部品を定義、return で結果を返す
  • タプル (x, y) とアンパック x, y = ... で座標を扱う
  • 「2回書きそうなら関数にする」がきれいなコードの鉄則
