五目並べ LESSON 02 / 06

石を置くしくみ

盤面はできました。次は「石を置く」関数と、「手番を交代する」しくみを作ります。オセロと違ってひっくり返しがない分、驚くほどシンプルに書けます。

🎯 このレッスンのゴール

  • 座標 (x, y) を指定して石を置く place 関数を作る
  • すでに石があるマスには置けないようにする
  • 手番を交代する opponent を使い、交互に石を置く流れを体験する

STEP 1座標のおさらい

五目並べでもオセロと同じ座標系を使います。x は横方向(列)、y は縦方向(行)。盤面への読み書きは board[y][x](yが先)でしたね。

STEP 2place関数を作る

「すでに石があるマスには置けない」——これが五目並べで唯一守るべきルールです。置けたら True、置けなければ False を返すようにしておくと、あとでゲームループを作るときに便利です。

ちなみにオセロの place は「ひっくり返せる石があるか」まで判定する複雑な関数になります。五目並べは「空いているか」だけを見ればいいので、たった4行。ルールが単純なゲームは、コードも単純になる——プログラミングあるあるです。

STEP 3手番を交代しながら置く

opponent でBLACKとWHITEを交互に切り替えながら、何手か置いてみましょう。

黒石が斜めに3つ並んでいるのが見えるはずです。あと2つ、同じ斜め線上に置ければ五目並べの勝ちになります——その「並びを見抜く」判定を、次のレッスンで作ります。

チャレンジ:place と opponent を自分の手で

このレッスンの主役2つ、place と opponent を、上のコードをなるべく見ずに下の専用エディタへ書けるか挑戦してみましょう。ポイントは「すでに石があるマスには置けない」——False が返って、盤面が変わらないことです。実行すると複数のテストデータで自動チェックされます。

解答例を見る 
def place(board, x, y, color):
    """(x, y) が空いていれば color を置いて True。すでに石があれば False"""
    if board[y][x] != EMPTY:
        return False
    board[y][x] = color
    return True

def opponent(color):
    return WHITE if color == BLACK else BLACK

board = make_board()
print(place(board, 4, 4, BLACK))   # True
print(place(board, 4, 4, WHITE))   # False(もう石がある)
print(opponent(BLACK))             # ○
print_board(board)

「置けたかどうかを True / False で返す」設計のおかげで、あとで作るゲームループが「置けなければ選び直す」と自然に書けるようになります。この2つは次のレッスンから自動で読み込まれる部品になるので、自分の手で書けるようにしておきましょう。

STEP 4まとめ

  • place は「空いていれば置く・すでにあれば置かない」だけのシンプルな関数
  • opponent でBLACK/WHITEを交互に切り替える
  • 五目並べのコードはオセロより短いが、考え方の骨組みは同じ
