部品はすべて手の中に。手番の交代、パス、終了判定、勝敗発表——「ゲームループ」を組み上げて、最初の1局を最後まで走らせます。

🎯 このレッスンのゴール

  • ゲーム全体の流れ(ゲームループ)を設計できる
  • パスと終了(両者連続パス)を正しく処理する
  • 1局を自動対局させて、勝敗を発表する

STEP 1ゲームの流れを言葉で書く

コードを書く前に、オセロ1局の流れを日本語で整理します。プログラム設計の第一歩は、いつもこれです。

1. 盤面を初期化。手番は黒から
2. ゲームが終わるまでくり返し:
     a. いまの手番が打てる場所を探す
     b. あれば → どこかに打つ(パス連続回数をリセット)
        なければ → パス(パス連続回数 +1)
     c. 手番を交代
3. 「両者連続パス」になったら終了
4. 石を数えて勝敗発表

「打つ場所をどう選ぶか」だけがまだ決まっていませんね。今回はランダムに選ぶことにします(次回、ここが「AI」に進化します)。今日のゴールは、こんな終局図にたどり着くことです。

STEP 2手番の交代とrandom

2つの新しい道具を確認しておきましょう。手番交代は opponent を自分に使うだけ。ランダム選択は標準ライブラリ random の出番です。

STEP 3ゲームループを組む

いよいよ組み立てです。STEP 1の日本語を、1行ずつPythonに翻訳していきます。passes(パスの連続回数)が2になったら終了、がポイントです。

実行するたびに違う棋譜が流れて、最後まで打ち切った盤面が表示されます。60手前後の対局が一瞬で終わる——コンピュータの計算力を体感できたのではないでしょうか。

show_moves=False のような「デフォルト値つき引数」を使うと、play_game() と呼べば静かに、play_game(show_moves=True) と呼べば実況つきに、と切り替えられます。

STEP 4勝敗を発表する

最後の仕上げ、結果発表です。count_stones で数えて比べるだけ。

これで、あなたはオセロを1本作り上げました。 make_board から始まった部品たちが、1つのゲームとして動いています。

チャレンジ:100局トーナメント

ランダム同士だと黒と白、どちらが勝ちやすいのでしょう? play_game() を100回くり返して、黒勝ち・白勝ち・引き分けの回数を集計してみましょう(数秒かかります)。

解答例を見る 
black_wins = 0
white_wins = 0
draws = 0

for i in range(100):
    board = play_game()
    b = count_stones(board, BLACK)
    w = count_stones(board, WHITE)
    if b > w:
        black_wins += 1
    elif w > b:
        white_wins += 1
    else:
        draws += 1

print(f"黒 {black_wins} 勝 / 白 {white_wins} 勝 / 引き分け {draws}")

何度か実行すると、毎回結果が揺れつつも大きくは偏らないことが分かります。この「大量対局で実力を測る」手法は、次回のAI開発でフル活用します。

STEP 5まとめ

  • ゲームループ = 「打てるなら打つ / 打てないならパス / 交代」のくり返し
  • 終了条件は「連続2回のパス」。passes カウンタで管理
  • デフォルト引数で関数の動きを切り替えられる
