STEP 1ゲームの流れを言葉で書く

コードを書く前に、オセロ1局の流れを日本語で整理します。プログラム設計の第一歩は、いつもこれです。

1. 盤面を初期化。手番は黒から
2. ゲームが終わるまでくり返し:
     a. いまの手番が打てる場所を探す
     b. あれば → どこかに打つ(パス連続回数をリセット)
        なければ → パス(パス連続回数 +1)
     c. 手番を交代
3. 「両者連続パス」になったら終了
4. 石を数えて勝敗発表

「打つ場所をどう選ぶか」だけがまだ決まっていませんね。今回はランダムに選ぶことにします(次回、ここが「AI」に進化します)。今日のゴールは、こんな終局図にたどり着くことです。

STEP 2手番の交代とrandom

2つの新しい道具を確認しておきましょう。手番交代は opponent を自分に使うだけ。ランダム選択は標準ライブラリ random の出番です。

Pythonには最初から使える道具箱(モジュール)がたくさん付属しています。import random で「乱数の道具箱」を読み込むと、random.choice(リスト)(=リストから1つ無作為に選ぶ)のように道具箱名.道具名の形で使えます。import はファイルの先頭に書くのが習慣です。

STEP 3ゲームループを組む

いよいよ組み立てです。STEP 1の日本語を、1行ずつPythonに翻訳していきます。passes(パスの連続回数)が2になったら終了、がポイントです。

実行するたびに違う棋譜が流れて、最後まで打ち切った盤面が表示されます。60手前後の対局が一瞬で終わる——コンピュータの計算力を体感できたのではないでしょうか。ちなみに手数が縦にきれいにそろっているのは、Lesson 2でも登場した書式指定 {move_no:2d}(2桁幅の整数で表示)のおかげです。

show_moves=False のような「デフォルト値つき引数」を使うと、play_game() と呼べば静かに、play_game(show_moves=True) と呼べば実況つきに、と切り替えられます。

STEP 4勝敗を発表する

最後の仕上げ、結果発表です。count_stones で数えて比べるだけ。

これで、あなたはオセロを1本作り上げました。 make_board から始まった部品たちが、1つのゲームとして動いています。

チャレンジ:トーナメントの集計係

ランダム同士だと黒と白、どちらが勝ちやすいのでしょう? それを調べるには、何局も対戦させて勝敗を集計する係が必要です。対戦結果 (黒の石数, 白の石数) のリストを受け取って、(黒勝ち, 白勝ち, 引き分け) の回数を返す関数 tally(results) を、下の専用エディタで完成させましょう。実行すると手作りの対戦データで自動チェックされ、おまけに本物の10局トーナメントも回ります。

解答例を見る 
import random

def play_game():
    """ランダム同士で1局対戦し、(黒の石数, 白の石数) を返す"""
    board = make_board()
    turn = BLACK
    passes = 0
    while passes < 2:
        moves = valid_moves(board, turn)
        if moves:
            x, y = random.choice(moves)
            place(board, x, y, turn)
            passes = 0
        else:
            passes += 1
        turn = opponent(turn)
    return count_stones(board, BLACK), count_stones(board, WHITE)

def tally(results):
    """(黒の石数, 白の石数) のリストを集計して (黒勝ち, 白勝ち, 引き分け) を返す"""
    black_wins = 0
    white_wins = 0
    draws = 0
    for b, w in results:
        if b > w:
            black_wins += 1
        elif w > b:
            white_wins += 1
        else:
            draws += 1
    return black_wins, white_wins, draws

results = []
for i in range(10):
    results.append(play_game())

b, w, d = tally(results)
print("対戦結果:", results)
print(f"黒 {b} 勝 / 白 {w} 勝 / 引き分け {d}")

for i in range(10) の 10 を 100 に変えれば100局トーナメントになります(数秒かかります)。何度か実行すると、毎回結果が揺れつつも大きくは偏らないことが分かります。この「大量対局で実力を測る」手法は、次回のAI開発でフル活用します。
