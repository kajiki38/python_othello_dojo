AIをもっと強く

最終レッスン。「マスの価値」と「評価関数」という本格AIの考え方を導入して、欲張りAIを倒します。ここから先は、あなた自身の研究テーマです。

🎯 このレッスンのゴール

  • 「マスごとの価値」を重みテーブルで表現する
  • 盤面のコピーと「試し打ち」で1手先を評価する
  • 評価関数ベースのAIを作り、欲張りAIとの対戦で検証する
  • 「序盤ランダム」でAIの強さをフェアに測る方法を知る

このページでは、これまで作った全部品に加えて ai_random / ai_greedy / play_match も読み込み済みです。

STEP 1マスには「価値」がある

前回、欲張りAIが意外と勝ちきれないことを見ました。理由はマスの価値がまるで違うから。角は絶対に取り返されない最強のマス。逆に角のナナメ隣(X打ちと呼ばれます)は、相手に角を献上しがちな危険地帯です。

この知識を、8×8の重みテーブルとしてプログラムに教え込みます。

数字は「そのマスに自分の石があると、どれくらい嬉しいか」。角は+100、X打ちは−50。この値はオセロAI研究で昔から使われている定番の形ですが、正解はありません。あとで自由に調整してみてください。

STEP 2試し打ち — 盤面をコピーして1手先を見る

「この手を打ったら盤面がどうなるか」を知るには、コピーした盤面で試し打ちすればOK。本物の盤面は汚さずに未来を覗けます。

test = board ではコピーになりません! 同じ盤面に2つの名前がつくだけで、test を変えると board も変わってしまいます。リストのコピーには row[:](スライスコピー)のような明示的な複製が必要——Python学習者が全員一度はハマる落とし穴です。

STEP 3評価関数AI — ai_master 誕生

部品が揃いました。「各手を試し打ちして、重みの合計点が一番高くなる手を選ぶ」AIを作ります。盤面の点数を計算する関数を評価関数と呼びます。

「まだ無い」の目印に None: best_score の初期値を 0 ではなく None(=まだ何も入っていない)にしているのがポイントです。評価点はマイナスにもなり得るので、0 から始めると「0より低いけど最善」の手を取り逃します。best_score is None(=まだ1手も比べていない)なら無条件に採用し、2手目以降は点数で比べる——「最初の1個は必ず採用する」ときの定番パターンです。

おそらく圧勝ですね。「目先の石数」より「場所の価値」——たった8×8の数字テーブルで、AIは見違えるほど強くなります。

…ところで、2局の結果がまったく同じだったことに気づきましたか?

STEP 4強さを正しく測る — 決定的AIの落とし穴

理由はこうです。ai_master も ai_greedy もランダム要素ゼロ。同じ盤面では必ず同じ手を選ぶので、何局やってもまったく同じ棋譜をなぞるだけなのです。つまりさっきの勝利は「その1つの展開で勝てた」ことしか証明していません。100局やっても情報量は1局分!

解決策はシンプルで、対局のスタート地点をバラけさせること。最初の数手だけランダムに打ち、そこから先を実力勝負にします。プロのオセロAI開発でも使われている定番の測定方法で、プレイグラウンドのAI工房もこの方式です。

今度は毎回違う結果になり、それでも ai_master が高い勝率をキープするはず。これが本当の実力差です。「先手・後手の交代」と「序盤ランダム」——この2つがAIの強さ測定の基本です。

このページのウィジェットには、STEP 3までで作った ai_master 一式も読み込み済みです(📦マーク)。だからこのエディタでは ai_master をそのまま呼べています。

STEP 5研究テーマ — ここからは自由研究

ここから先に「教科書」はありません。強いオセロAIづくりの定番アイデアをいくつか紹介するので、好きなものを選んで ai_master を改造してみてください。

  • 重みの調整 — WEIGHTSの数字を変えて対戦実験。角は100で足りる? X打ちはもっと減点?
  • 着手可能数(モビリティ) — Lesson 4で作った「打てる場所の数」を評価に足す。evaluate に + 3 * len(valid_moves(board, color)) のような項を追加
  • 終盤モード — 残りマスが少なくなったら(例: 12マス以下)、重みではなく石数そのもので評価を切り替える
  • 2手先読み — 自分の手の後、相手が最善手を打ったときの評価で選ぶ(ミニマックス法への入り口)

ラスボスチャレンジ:モビリティ入りAI

「重み + 自分の着手可能数×3 − 相手の着手可能数×3」で盤面を採点する evaluate2 と、それで手を選ぶ ai_ultimate を、下の専用エディタで完成させましょう(ai_master の形がそのままお手本になります)。実行するといろいろな盤面で自動チェックされます。合格したら STEP 4 のエディタに持ち込んで、play_match_varied で ai_master と30局対戦させてみましょう(決定的AI同士の対決だからこそ!)。

解答例を見る 
def evaluate2(board, color):
    """重みの点数に「着手可能数の差 × 3」を足した新しい評価関数"""
    score = evaluate(board, color)   # 重みによる点数
    my_mob = len(valid_moves(board, color))
    op_mob = len(valid_moves(board, opponent(color)))
    return score + 3 * my_mob - 3 * op_mob

def ai_ultimate(board, color, moves):
    """全部の手を試し打ちして、evaluate2 が最高の手を選ぶAI"""
    best_move = None
    best_score = None
    for x, y in moves:
        test = copy_board(board)
        place(test, x, y, color)
        score = evaluate2(test, color)
        if best_score is None or score > best_score:
            best_score = score
            best_move = (x, y)
    return best_move

board = make_board()
moves = valid_moves(board, BLACK)
print("初期盤面の評価:", evaluate2(board, BLACK))
print("ai_ultimate の選択:", ai_ultimate(board, BLACK, moves))

合格したら STEP 4 のエディタで play_match_varied(ai_ultimate, ai_master) の30局対戦へ。係数「3」を変えると強さはどう変わるでしょう? こうした実験の積み重ねが、AI開発の醍醐味です。
