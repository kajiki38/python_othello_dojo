発展: 3手先を読むAI

前回完成した ai_strong も、実は「打った直後」しか見ていません。今回はどのゲームにも通じる汎用アイデアミニマックスで、相手の返し・そのまた先まで読むAIを作ります。コネクトフォーは1手がたった7通り——分岐が少ないゲームは深く読めるのが最大の武器。オセロや五目並べなら2手先が現実的なところ、ここでは3手先まで読みます。ミニマックスの見せ場です。

🎯 このレッスンのゴール

  • 「1手先読み」の限界 — 2手先に潜む罠を体感する
  • 盤面を点数にする評価関数と、「相手は一番イヤな返しをしてくる」という読みの心(max と min)を知る
  • 再帰(自分を呼ぶ関数)で minimax_score を書き、3手先を読む ai_minimax を完成させる

STEP 1「1手先」の限界

ai_strong は「勝ち → 防御 → ダブルリーチ → 先回り → 安全な中央」と、かなり慎重に手を選びます。safe_columns で相手の返しも1手分ケアしていますが、チェックしているのは「落とした直後に即勝ちされないか」だけ。2手先に仕掛けられた罠は見えません。実際の局面で確かめましょう。赤も黄も ai_strong、赤の番です。

赤が中央の3列目に落とした瞬間、その石が台になりました。黄がすかさず上に乗って4段目に○○○——勝ち列が [1, 5] の2つ、ダブルリーチ完成です。赤は片方しか塞げず、もう手遅れ。3列目に落とした時点では黄の即勝ちは存在しないので、safe_columns も④の先回りチェックもこの罠を検知できません。罠が姿を現すのは黄が乗った後=2手先——ai_strong の読みの外側です。「相手の返しの、そのまた先」まで読む仕組みが必要です。

STEP 2相手は一番イヤな返しをしてくる

先読みAIの部品は2つあります。部品①は、盤面の良し悪しを1つの数字にする評価関数。ここでは「勝ち列の数の差」を軸に、ダブルリーチはほぼ勝ち(±500)、おまけに中央列の石を1個1点で足します。

なんと、罠の3列目が最高点(中央ボーナス +1)です。「打った直後」の点数だけでは罠は見えない——STEP 1 の再確認になりました。そこで部品②です。3列目に落とした後、黄のすべての返しに点数を付けてみます。

黄が3列目に乗り返した瞬間だけ -500(ダブルリーチ)。黄はもちろん、こちらに一番イヤなこの返しを選びます。つまり「3列目に落とす手の本当の価値」は、返しの点数の最小値(min)= -500 です。逆に、自分が選ぶ番では点数が最大(max)の手を選ぶ。自分は max、相手は min——この交互の読みこそがミニマックスです。

STEP 3再帰 — 関数が自分を呼ぶ

「相手の返しの、そのまた先」まで読むにはどう書けばいいでしょう? コツは、「あと depth 手読む」を「1手進めて、あと depth−1 手読む」と言い換えることです。同じ形の小さい問題に置き換える——これは関数が自分自身を呼ぶと自然に書けます。まずは一番小さな例から。

関数が自分自身を呼ぶ書き方を再帰といいます。①必ず「止まる条件」を先に書く ②呼ぶたびに問題を小さくする(n-1、depth-1)——この2点さえ守れば怖くありません。ミニマックスは『depth 手先まで読む』を『1手進めて depth-1 手先まで読む』に置き換える、再帰の代表例です。

これをミニマックスに当てはめたのが minimax_score です。止まる条件は「depth == 0 か、落とせる列がない」——そこで評価関数が盤面を点数化します(読みの底)。途中で4連が完成したら、その場で ±1000 の決着点を返します。

あと1手読むと -500——STEP 2 で手作業でやった min の計算と同じ結果です。あと3手読むと -1000。「黄がダブルリーチを作り、赤が片方を塞いでも、黄が残りで勝ち切る」という結末まで読み切って、必敗と言い切りました。1列目なら -2 でセーフ。depth を増やすほど、遠くの罠まで見えるのです。

オセロ版と違って盤をコピーせず、試しに置いて必ず戻す方式なのもポイント(Lesson 5・6 でおなじみの型ですね)。コピーが要らないぶん高速です。そして注目してほしいのはコストです。コネクトフォーの分岐は1手あたり最大7通り。3手読んでも 7×7×7 = 343 通りほどしかありません。分岐が10以上あるオセロや、候補を絞っても盤の広い五目並べでは2手先が現実的なところ、コネクトフォーはもっと深く潜れます。

STEP 4ai_minimax 完成 — そして対戦実験

仕上げは簡単。落とせる各列を試し、minimax_score の点数が最大の列を選ぶだけです。depth=3 は「自分 → 相手 → 自分」の3手ぶんを読む設定です。

ai_strong がはまった3列目の罠を、ai_minimax はきれいに回避しました。しかも特別なコネクトフォーの知識(ダブルリーチ探し、うっかり手回避…)は一切書いていません。評価関数とミニマックスだけで、罠が「点数の低い手」として自然に浮かび上がるのです。それでは対戦実験。ai_block と20局、ai_strong と20局です(1局ごとに先手後手交代)。

筆者の実測(各50局)では ai_block に45勝4敗1分——9割勝てる圧勝です。一方 ai_strong には10勝19敗21分。引き分けがとても多い、がっぷり四つの接戦です。コネクトフォー専用テクニックを磨き込んだ ai_strong はさすがに手強い——それでも、どのゲームにも使える汎用アイデア1本でここまで並んだこと自体がすごいことです。速度は1手 約9ms(depth=3)とまだまだ余裕。そう、コネクトフォーには深さを上げるという伸びしろが残っています。追い抜くのはこの後のチャレンジで!

チャレンジ:もっと深く読め — depth=4

分岐が7通りしかないコネクトフォーなら、まだ深く潜れます。ai_minimax の depth を 4 に上げた ai_minimax2 を完成させましょう。minimax_score はそのまま使えます——ai_minimax を depth 指定で呼ぶだけです。スターターの盤面は、実は3列目以外に落とすと4手以内に必敗という追い詰められた局面。depth=3 のままだと罠が見えず2列目を選んでしまいますが、4手先まで読めれば唯一の生還ルートを見つけられます。実行すると、この局面に加えてランダムな局面(合法手・勝ち切り・ブロック)で自動チェックされます。

def evaluate(board, color):
    opp = opponent(color)
    my = len(winning_cols(board, color))
    op = len(winning_cols(board, opp))
    if my >= 2:
        return 500
    if op >= 2:
        return -500
    center = 0
    for y in range(HEIGHT):
        if board[y][WIDTH // 2] == color:
            center += 1
        elif board[y][WIDTH // 2] == opp:
            center -= 1
    return my * 10 - op * 10 + center

def minimax_score(board, color, turn, depth):
    cols = valid_columns(board)
    if depth == 0 or not cols:
        return evaluate(board, color)
    scores = []
    for col in cols:
        y = next_row(board, col)
        board[y][col] = turn
        if check_win(board, col, y, turn):
            s = 1000 if turn == color else -1000
        else:
            s = minimax_score(board, color, opponent(turn), depth - 1)
        board[y][col] = EMPTY
        scores.append(s)
    if turn == color:
        return max(scores)
    return min(scores)

def ai_minimax(board, color, cols, depth=3):
    best_col, best_score = None, None
    for col in cols:
        y = next_row(board, col)
        board[y][col] = color
        if check_win(board, col, y, color):
            s = 1000
        else:
            s = minimax_score(board, color, opponent(color), depth - 1)
        board[y][col] = EMPTY
        if best_score is None or s > best_score:
            best_col, best_score = col, s
    return best_col

def ai_minimax2(board, color, cols):
    return ai_minimax(board, color, cols, depth=4)   # 深さを 3 → 4 に上げるだけ!

再帰で書いておいたおかげで、読みを深くする改造は引数を1つ変えるだけ。これが構造のきれいなコードの威力です。ただし深さのコストは指数的に増えます——実測では depth=4 で1手 約60ms(まだ快適)、depth=5 だと約400ms とかなり待たされます。強さと速度のトレードオフを自分で選べるのも、ミニマックスの面白いところです。

合格したら、STEP 4 の実験コードの ai_minimax を ai_minimax2 に替えて ai_strong と20局対戦してみましょう(1手が重いので数十秒かかります)。筆者の実測では 8勝10敗2分——depth=3 でははっきり負け越していた相手と、ほぼ互角まで来ました。さらに上を目指すなら evaluate の改造(リーチの重み、中央ボーナスの調整)です。AI工房で自分だけの最強AIに磨き上げてください!

STEP 5まとめ — どのゲームでも同じ形

  • 1手先しか見ないAIは、2手先に仕掛けられた罠にはまる
  • 評価関数で盤面を点数に。手の価値は「自分は max、相手は min」の交互の読み=ミニマックスで決まる
  • 再帰のコツは2つ——止まる条件(depth == 0)を先に書く、呼ぶたびに問題を小さくする(depth - 1)
  • 分岐がたった7通りのコネクトフォーは深読みの見せ場。depth=3 で1手 約9ms、depth=4 でも実用速度

「評価関数+ミニマックス」という骨組みは、オセロでも五目並べでも、将棋やチェスでもまったく同じ形です。ゲームごとに変わるのは評価関数と手の生成だけ——つまり今日あなたが書いたのは、どのゲームにも持っていける一生モノの武器です。おつかれさまでした! AI工房で depth や evaluate を調整して、自分だけの最強AIを育ててください。
