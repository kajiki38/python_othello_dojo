五目並べ LESSON 05 / 06

対戦AIを作る

五目並べ道場、最終レッスンです。「勝てる手があれば勝つ」「相手が勝ちそうなら防ぐ」——たった2つのルールで、ランダムAIよりずっと強いAIが作れます。

🎯 このレッスンのゴール

  • AI = 「盤面を見て手を返す関数」という考え方(この道場に共通の型)
  • 「試しに置いてみて判定し、元に戻す」というテクニックを使う
  • 勝ち・防御を読む ai_block を作り、ランダムAIと対戦実験する

STEP 1ランダムAI(基準点)

まずは比較の基準になるランダムAIから。この道場共通のインターフェース——(board, color, moves) を受け取って1手返す関数です。

STEP 2「試しに置いてみる」テクニック

「この手を打ったら勝てるか?」を調べる一番簡単な方法は、実際に試しに置いてみてcheck_winで判定し、ダメならすぐ元に戻すことです。盤面を1回だけ書き換えて調べる、軽量なやり方です。

元に戻し忘れに注意! board[y][x] = EMPTY を忘れると、調べただけのはずの石が盤面に残ってしまいます。「試しに置く→調べる→戻す」は3点セットで覚えましょう。

STEP 3勝ち + 防御 — ai_block

find_winning_move は「自分の勝ち手を探す」だけでなく、「color に相手の色を渡せば、相手の勝ち手=自分が防ぐべき手」も探せます。同じ部品を2通りに使い回すのがポイントです。

STEP 4実験:防御AI vs ランダムAI

AIの強さは大量対局の勝率で測るのがこの道場の流儀。対戦させる装置 play_match を作って50局戦わせましょう。

{wins / 50 * 100:.0f} の :.0f はf-stringの書式指定で、「小数点以下は表示しない(四捨五入)」という意味。84.0% ではなく 84% とすっきり表示できます。

関数を「渡す」— 初めての大技: play_match(ai_block, ai_random) の呼び出しに注目してください。ai_block() のように実行するのではなく、ai_block という関数そのものを引数として渡しています。play_match の中では、受け取った関数を ai = ai_black if turn == BLACK else ai_white で手番に応じて選び、ai(board, turn, moves) の形で呼び出します。Pythonでは関数も値の一種なので、こうして「どのAIで戦わせるか」をあとから差し替えられるのです。この仕組みのおかげで、次のレッスンで作るもっと強いAIも、同じ play_match にそのまま渡して試せます。

「勝てる手を逃さず、負けそうな手だけ防ぐ」——たったこれだけで、ランダムAIにかなりの差をつけられるはずです。五目並べは読みの深さがそのまま強さに直結するゲームなので、この find_winning_move の考え方をさらに2手先・3手先まで広げれば、もっと強いAIに育てられます。

チャレンジ:自分だけの改造AI

ai_block をベースに、「勝ち・防御どちらもなければ、盤の中心に近い手を選ぶ」ai_center を、下の専用エディタで完成させましょう。五目並べは中心付近ほど並びを作りやすい、というのが定石です。ランダム要素がなくなるので、実行するといろいろな盤面で「勝ち手を逃さないか」「相手のリーチを防げるか」「中央に寄れるか」が自動でチェックされます。

解答例を見る 
def find_winning_move(board, color, moves):
    for x, y in moves:
        board[y][x] = color
        win = check_win(board, x, y, color)
        board[y][x] = EMPTY
        if win:
            return (x, y)
    return None

def ai_center(board, color, moves):
    win_move = find_winning_move(board, color, moves)
    if win_move:
        return win_move
    block_move = find_winning_move(board, opponent(color), moves)
    if block_move:
        return block_move
    center = SIZE // 2
    # 中心からの距離が一番近い手を選ぶ
    return min(moves, key=lambda m: (m[0] - center) ** 2 + (m[1] - center) ** 2)

board = make_board()
for x, y in [(2, 2), (3, 3), (4, 4), (5, 5)]:
    place(board, x, y, WHITE)
print_board(board)
print("黒番の ai_center の選択:", ai_center(board, BLACK, empty_cells(board)))

key=lambda m: 式 は「各手 m をいったん式の値に変換してから比べる」という指定です。ここでは (m[0] - center) ** 2 + (m[1] - center) ** 2(中心からの距離。** 2 は「2乗」)で各手を点数化し、min がそのいちばん小さい=中心にいちばん近い手を選びます。lambda は「その場で作る、名前のない小さな関数」——今は『各要素を点数に変える式』と読めればOKです。min/max の key は、どの種目のAI作りでも使える定番テクニックなので形に慣れておきましょう。完成した ai_center を STEP 4 の play_match に渡して ai_random と50局戦わせれば、勝率でも強さを確かめられます。

STEP 5まとめ

  • 「試しに置く→check_winで判定→元に戻す」で、先読み1手のAIが作れる
  • 同じ部品(find_winning_move)を「攻め」と「守り」の両方に使い回せる
  • 強さは大量対局の勝率で測る

ところで——この ai_block、実は人間が簡単に勝てる弱点を抱えています。次の最終レッスンでその弱点を暴いて、1手先を読む強化AIに育てましょう。
