AIをもっと強く

前回の ai_block は、実は「底辺に3つ並べる」だけで人間に負けます。弱点の正体ダブルリーチと、コネクトフォー特有の「うっかり手」——2つの穴を先読みで塞いだ強化AIを作ります。

🎯 このレッスンのゴール

  • 1手読みAIの弱点「ダブルリーチ」を理解する
  • 「落とすと相手に上で勝たれる列」= うっかり手を避ける
  • 勝ち列を数えて、ダブルリーチを「狙う」「先回りして防ぐ」AIを完成させる

STEP 1ai_block はなぜ負ける?

赤(相手)が底辺に両端の空いた3連を作った局面です。ai_block は片方の端を防げますが……

片側(1列)を塞いでも、反対側(5列)が残っていて手遅れ。勝ち列が2つある形(ダブルリーチ)は、できてしまったら防げません。防ぐタイミングは「3連が完成する前」——つまり1手先を読む必要があります。

STEP 2勝ち列を「数える」

find_winning_col は勝ち列を「1つ見つけたら終わり」でした。ダブルリーチを見抜くために、ぜんぶ集める版を作ります。

[1, 5] — リストの長さが2以上ならダブルリーチです。この数を使えば、必勝形を「作られる前に検知」できます。

STEP 3コネクトフォー特有の「うっかり手」

もうひとつ、このゲームならではの罠があります。自分が落とした石が「台」になって、相手がその上に乗って勝つパターンです。2段目に赤の3連が眠っている局面で確かめましょう。

0列と4列が消えました。黄がそこに落とすと、赤がその上(2段目)に乗って4連が完成してしまうからです。「打った直後の相手の返し」を1手だけ読む——これだけで、うっかり負けが激減します。

STEP 4強化AI ai_strong を組み立てる

部品がそろったので合体させます。優先順位は 勝ち → 防御 → (安全な列の中で)ダブルリーチを狙う → 相手のダブルリーチを先回りして塞ぐ → 中央寄り。

赤がまだ2連の段階で、黄は1列か4列 — 3連が両端オープンになる場所 — に先回りしました。「ここに落とされたら勝ち列が2つになる」と読んだからです。もうSTEP 1の必勝パターンは通用しません。

STEP 5対戦実験:先読みAI vs 勝ち・防御AI

仕上げに ai_block と50局対戦(先手後手は1局ごとに交代)。読みの深さ1手分の差を数字で確かめましょう。なお、ランダム要素のないAIどうしだと毎回まったく同じ棋譜になってしまうため、play_match は各対局の最初の2手をランダムに落としてから実力勝負に入る作りにしてあります。

チャレンジ:攻めをさらに鋭く

いまのAIは、ダブルリーチが作れないときは「中央寄りの安全な列」に置くだけ。「せめてリーチ(勝ち列1つ)を作れる列を優先する」よう改造した ai_strong2 を、下の専用エディタで完成させましょう。スターターには ai_strong と同じ中身が入っているので、③のダブルリーチ狙いを「2つ→なければ1つ」の2段構えに育てるのが仕事です。実行すると、勝ち・防御・ダブルリーチ・リーチ作り・うっかり手回避の各局面で自動チェックされます。

def winning_cols(board, color):
    wins = []
    for col in valid_columns(board):
        y = next_row(board, col)
        board[y][col] = color
        if check_win(board, col, y, color):
            wins.append(col)
        board[y][col] = EMPTY
    return wins

def safe_columns(board, color, cols):
    safe = []
    for col in cols:
        y = next_row(board, col)
        board[y][col] = color
        if not winning_cols(board, opponent(color)):
            safe.append(col)
        board[y][col] = EMPTY
    return safe

def ai_strong2(board, color, cols):
    win_col = find_winning_col(board, color, cols)
    if win_col is not None:
        return win_col
    block_col = find_winning_col(board, opponent(color), cols)
    if block_col is not None:
        return block_col
    center = WIDTH // 2
    cand = safe_columns(board, color, cols)
    if not cand:
        cand = cols
    cand = sorted(cand, key=lambda c: abs(c - center))
    for need in (2, 1):                  # まずダブルリーチ、なければ単リーチ
        for col in cand:
            y = next_row(board, col)
            board[y][col] = color
            n = len(winning_cols(board, color))
            board[y][col] = EMPTY
            if n >= need:
                return col
        if need == 2:                    # ダブルリーチ狙いの直後に防御を挟む
            for col in cand:
                y = next_row(board, col)
                board[y][col] = opponent(color)
                n = len(winning_cols(board, opponent(color)))
                board[y][col] = EMPTY
                if n >= 2:
                    return col
    return cand[0]

リーチを作る=相手に防御を強制する、ということ。攻め続ければ相手は自分の形を作れません。テストに合格したら、プリアンブルに入っている play_match(ai_strong2, ai_block) や、STEP 4の ai_strong を貼り足しての play_match(ai_strong2, ai_strong) で、本当に強くなったか計測してみましょう(勝率で確かめる習慣、もう身についていますね)。

STEP 6まとめ — 3種目コンプリート!

  • ダブルリーチ(勝ち列2つ)はできる前にしか防げない — だから1手先を読む
  • コネクトフォーは「自分の石が相手の台になる」うっかり手のケアが必須
  • 「勝ち→防御→狙う→先回り→安全第一」という優先順位の設計が、AIの強さを決める
