リストとくり返し

オセロ盤も、碁盤も、コネクトフォーの盤も、正体は「データの並び」= リストです。リストと、それを一気に処理するforループ——この2つがゲーム作りの土台になります。

🎯 このレッスンのゴール

  • リストを番号(インデックス)で読み書きし、内包表記などで新しく作れる
  • len / count / join / append などリストの道具を使える
  • for(回数が決まったくり返し)と while(条件が続く間のくり返し)が書ける

STEP 1リスト — データを並べて持つ

ゲーム盤の1行は「マスの並び」です。こういう「並んだデータ」を扱うのがリスト。この道場全体で一番大事な道具です。

超重要: リストの番号(インデックス)は 0から 始まります。row[3] は「先頭から数えて4番目」。ゲーム盤の座標も0始まりで扱うので、この感覚に慣れておきましょう。

STEP 2リストは書き換えられる

「石を置く」「石をひっくり返す」は、プログラム的にはリストの中身の書き換えです。もう体験してしまいましょう。

" ".join(row) は「リストの中身をスペースでつないで1つの文字列にする」テクニックです。盤面をきれいに表示するのに使います。row.count("●") のような「リストに聞く」書き方も、石数のカウントで大活躍します。

STEP 3くり返し — forループ

8マス、8行、64マス…ゲームは「同じ処理のくり返し」だらけです。同じ命令を64回も手で書くのは大変。そこで使うのが for ループ。リストの中身を1個ずつ取り出して、同じ処理をくり返すしくみです。

まずは「形」を覚えましょう。for の基本形はこうです。

for 変数 in リスト:
    くり返したい処理     # ← 行の頭をスペース4つ字下げする

読み方は「リストの中身を1つずつ 変数 に入れながら、字下げした処理を上から順にくり返す」。言葉で言うより、動かして見るのが早いです。

ここで cell がループ変数です。リストの中身が1個ずつ順番に cell に入り、リストを使い切ると自動的にループが終わります。頭の中では、こう動いています。

1周目: cell = "●"  →  print  →  いまのマスは ●
2周目: cell = "●"  →  print  →  いまのマスは ●
3周目: cell = "○"  →  print  →  いまのマスは ○
        （リストの終わり。ループ終了）

くり返しの回数を自分で数える必要はありません。リストの長さぶんだけ、Pythonが勝手にくり返してくれる——これが for の便利さです。

range() で「番号」をくり返す

もう1つ、よく使う形があります。中身ではなく「番号」がほしいとき。range(8) は 0, 1, 2, …, 7 という連番を順に生み出します(8個ぶん、0から始まって8は含まないのがポイント)。

今度はループ変数 i に 0 → 1 → 2 → … → 7 が順に入るので、row[i] で全マスを番号つきで読めます。使い分けはシンプルです。

  • 中身がほしいだけ → for cell in row:
  • 番号(何番目か)も使いたい → for i in range(len(row)):

コロンとインデントを忘れずに: for の行末には :(コロン)、くり返したい処理はスペース4つで字下げします。字下げした行だけがくり返しの中身です。逆に字下げを戻せば「ループの外」に出たことになります(上の例の print("ループが終わりました") は字下げしていないので、くり返されず最後に1回だけ実行されます)。コロンと字下げは、Pythonで最初につまずく定番ポイント。指に覚えさせましょう。

STEP 4ループで計算する

ループの中で変数を育てていくのも定番の型です。「合計を求める」で練習しましょう。

チャレンジ:一番いいスコアは?

5回分のスコアのリストから、いちばん大きいスコアを変数 best に入れて表示するプログラムを、下の専用エディタに書いてみましょう。ヒント: Pythonには max(scores) という関数があります。STEP 4のようにループで変数を育てて求める方法もぜひ考えてみてください。実行すると自動でチェックされます。

解答例を見る 
scores = [34, 41, 28, 55, 33]

best = 0
for s in scores:
    if s > best:      # if(条件分岐)は次のレッスンで詳しく!
        best = s

print(f"最高スコア: {best}")

best = max(scores) と書けば1行で求まりますが、「ループの中で最大を探す」型もこの先ずっと使うので、両方できるようにしておきましょう。ループ版に出てきた if は「もし〜なら」の条件分岐。次のレッスンの主役です。

STEP 5リストを作る・育てる

ここまではすでにあるリストを使ってきました。最後に、リストを一から作る方法を覚えましょう。ゲーム盤を用意するのに欠かせない道具です。

まず、同じ値を並べただけのリストは *(かけ算)で一気に作れます。そして空のリストに append で1個ずつ足して育てられます。

["."] * 8 は「"." を8個ぶん並べたリスト」。リスト.append(値) は「リストの末尾に値を1つ足す」命令です。空リスト [] から始めて append で育てる——これはこの先ずっと使う型です。

では、ゲーム盤のような「リストのリスト」(2次元)はどう作るか。「1行」を必要な数だけ積み重ねます。

この「空き行を8回作って積む」処理は、じつは1行でも書けます。次の書き方はリスト内包表記と呼ばれ、この道場のあらゆる盤面づくりで登場する定番です。

[ ["."]*8  for _ in range(8) ] は「["."]*8(=.が8個の行)を range(8) のぶん(8回)くり返し作って、1つのリストにまとめる」という意味です。ここで _(アンダースコア)は「使わない変数」の慣習的な名前。for i in range(8) の i を本体で使わないときに、_ にして「この番号は使いません」と示します。

内包表記は最初こそ読みにくく感じますが、「for を後ろに回した書き方」と思えば大丈夫。この先の各レッスンで make_board という関数の中に必ず出てくるので、形だけでも見慣れておきましょう。

STEP 6回数が決まらないくり返し — while

for は「リストのぶん」「range のぶん」と回数が決まったくり返しでした。もう1つ、「ある条件が成り立つ間ずっと」くり返す while があります。何回で終わるか事前に分からないときに使います。

while 条件: は「条件が成り立つ間、中身をくり返す」。1周ごとに条件をチェックし、成り立たなくなった瞬間に抜けます。上の例なら stones が5個未満になった時点で止まります。コツは、ループの中で条件に関わる変数(ここでは stones)を必ず変化させること。変化させ忘れると条件がずっと成り立ったままで、終わらない「無限ループ」になってしまいます。

ゲームでは「盤の端に着くまで1マスずつ進む」「勝敗が決まるまで手を打ち続ける」のように、終わりが状況しだいの場面で while が活躍します。とくに各種目の「石の並びを数える」処理で必ず登場するので、しっかり押さえておきましょう。なお、ループを途中でやめたくなったら break(ループをその場で打ち切って外へ抜ける命令)も使えます。種目コースで実際に登場します。

STEP 7まとめ

  • リストは [番号] で読み書きできる。番号は 0始まり
  • len(長さ)/ count(数える)/ join(つなぐ)/ max(最大)
  • for cell in row: と for i in range(8): の2つの形。コロンとインデントを忘れずに
  • リスト作り: ["."] * 8(並べる)/ append(末尾に足す)/ [[...] * 8 for _ in range(8)](2次元を1行で)
  • while 条件: は回数が決まらないくり返し。変数の変化を忘れず(無限ループ注意)

次のレッスンは条件分岐(if)。「勝ったのはどっち?」「そのマスは盤の中?」——ゲームの判断はぜんぶ if でできています。
